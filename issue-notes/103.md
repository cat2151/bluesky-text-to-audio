# issue effect トラックを実装する #103
[issues #103](https://github.com/cat2151/bluesky-text-to-audio/issues/103)

- 用途、YM2151や、Surge XTに、Tone.jsのPingPongDelayをかけて楽しむ用。小さく始める用。
  - 検証したら、別issueで、Surge XTのeffect等も想定して検討して検証を進めていく。
- イメージ、各trackをprerenderするところまでは既存と同じ。そのあと、formatに従って、当該のeffectをかけたものをさらにprerenderして、それをplayする。
- 課題
  - 機能不足。小さく始める場合、全体に1つの小さなeffectをかける、となる。対策、小さく始めるので、まずそれでよい。
  - Surge XTのeffectは、まだserver側が未実装なので使えない。対策、READMEに書いておくレベルの後回しとする。
- 対策案
  - まず小さく始める。破綻するようなら、READMEの今後の課題の場所に入れて、issue管理から外す。

### formatの具体例

```
YM2151 cde;
effect @PingPongDelay
```

- これは検証用の仮formatであり、「effect」という接頭辞は仮であり、別issueで破壊的変更をしていく考え。
- 上記をmixモードで演奏すると、
  - cdeがprerenderされ、efgがprerenderされ、それらがmixされて、audio dataが生成される。ここまでは既存と同じ
  - そのaudio dataをblobにし、createObjectURLして、blob URLを得る
  - そしてTone.js用MMLを生成する。以下である。これを従来のTone.js playと同様の変換をしてJSONにしてtonejs-json-sequencerでplayする。
  - 小さく始める。よって、`t60 l1 c` という、4秒間固定の演奏として検証する。

```
@Sampler{ "urls": { "C4": "ここにblob URLを書く" } } @PingPongDelay t60 l1 c
```
